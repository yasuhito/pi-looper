# 機能: 公開設定は安全な既定値と明確な優先順位で解決される

deadloop を導入する利用者が、設定元と共有方針のどちらが使われるか、危険な自動化が意図せず有効にならないかを予測できるようにする。

## シナリオ: 環境変数で指定した設定ファイルを使う

* 前提 三つの設定ファイルがある
* かつ `DEADLOOP_CONFIG` で設定ファイルを指定する
* もし deadloop が設定ファイルを探す
* ならば 指定した設定ファイルが選ばれる

## シナリオ: 利用者設定を同梱設定より優先する

* 前提 三つの設定ファイルがある
* かつ `DEADLOOP_CONFIG` を指定しない
* もし deadloop が設定ファイルを探す
* ならば 利用者設定ファイルが選ばれる

## シナリオ: 利用者設定がない場合は同梱設定を使う

* 前提 同梱設定ファイルだけがある
* もし deadloop が設定ファイルを探す
* ならば 同梱設定ファイルが選ばれる

## シナリオ: 設定を省略した場合は標準の自動化を使う

* 前提 空のプロジェクト設定がある
* もし deadloop がプロジェクト設定を解決する
* ならば 標準の自動化が有効になる

## シナリオ: 空の自動化設定は自動化を無効のままにする

* 前提 空の自動化設定がある
* もし deadloop がプロジェクト設定を解決する
* ならば 有効な自動化はない

## シナリオ: 設定を省略した Worker は pi を使う

* 前提 空のプロジェクト設定がある
* もし deadloop がプロジェクト設定を解決する
* ならば Worker は pi を使う

## シナリオ: 設定を省略した Reviewer は pi を使う

* 前提 空のプロジェクト設定がある
* もし deadloop がプロジェクト設定を解決する
* ならば Reviewer は pi を使う

## シナリオ: 指定した Worker エージェントを使う

* 前提 Worker に claude を指定したプロジェクト設定がある
* もし deadloop がプロジェクト設定を解決する
* ならば Worker は claude を使う

## シナリオ: 指定した Reviewer エージェントを使う

* 前提 Reviewer に claude を指定したプロジェクト設定がある
* もし deadloop がプロジェクト設定を解決する
* ならば Reviewer は claude を使う

## シナリオ: 指定した Worker モデルをそのまま使う

* 前提 Worker モデルを指定したプロジェクト設定がある
* もし deadloop がプロジェクト設定を解決する
* ならば 指定した Worker モデルが使われる

## シナリオ: 指定した Reviewer モデルをそのまま使う

* 前提 Reviewer モデルを指定したプロジェクト設定がある
* もし deadloop がプロジェクト設定を解決する
* ならば 指定した Reviewer モデルが使われる

## シナリオ: ローカルで省略した Worker モデルを共有方針が補う

* 前提 Worker モデルを含む共有方針と空のローカル設定がある
* もし deadloop がプロジェクト設定を解決する
* ならば 共有方針の Worker モデルが使われる

## シナリオ: ローカルの Worker モデルは共有方針より優先する

* 前提 異なる Worker モデルを含む共有方針とローカル設定がある
* もし deadloop がプロジェクト設定を解決する
* ならば ローカルの Worker モデルが使われる

## シナリオ: 共有方針はローカルで省略した指示ファイルを補う

* 前提 指示ファイルを含む共有方針と空のローカル設定がある
* もし deadloop がプロジェクト設定を解決する
* ならば 共有方針の指示ファイルが使われる

## シナリオ: 共有方針の空の自動化設定は自動化を無効のままにする

* 前提 空の自動化設定を含む共有方針と空のローカル設定がある
* もし deadloop がプロジェクト設定を解決する
* ならば 有効な自動化はない

## シナリオ: 共有方針はローカルで省略した自動化を補う

* 前提 自動化を含む共有方針と空のローカル設定がある
* もし deadloop がプロジェクト設定を解決する
* ならば 共有方針の自動化が有効になる

## シナリオ: 共有方針はローカルで省略した外部レビュー設定を補う

* 前提 外部レビューを含む共有方針と空のローカル設定がある
* もし deadloop がプロジェクト設定を解決する
* ならば 外部レビューは有効である

## シナリオ: 自動マージは明示した場合だけ有効になる

* 前提 自動マージを有効にしたプロジェクト設定がある
* もし deadloop がプロジェクト設定を解決する
* ならば 自動マージは有効である

## シナリオ: 設定を省略した場合は自動マージしない

* 前提 空のプロジェクト設定がある

## シナリオ: 設定を省略した場合は CI 代替を使わない

* もし deadloop がプロジェクト設定を解決する
* ならば CI 代替は無効である

## シナリオ: 設定を省略した場合は外部レビューを使わない
